## 🔄 完整運作流程

### 流程 1：PDF 萃取 (Tab 1)

```
用戶輸入
  ├─ GCP Project ID: cdcda-lab-377808
  ├─ Model ID: claude-sonnet-4-5@20250929
  ├─ Provider: claude (自動判斷)
  └─ Location: us-central1
         │
         ↓
    建立 ModelConfig
         │
         ↓
    初始化 PDFExtractor
         │
         ├─ Provider = "gemini" → 初始化 genai.Client
         └─ Provider = "claude" → 初始化 AnthropicVertex
         │
         ↓
    掃描 samplepdflist/ 目錄
         │
         ↓
    逐一處理 PDF 檔案
         │
         ├─ 1. 讀取 PDF 檔案
         ├─ 2. 轉換為純文字
         ├─ 3. 呼叫 LLM API
         │    ├─ Gemini: genai_client.models.generate_content_stream()
         │    └─ Claude: claude_client.messages.create()
         ├─ 4. 解析 JSON 回應
         ├─ 5. 驗證資料格式
         └─ 6. 添加 metadata (CASE_LINK, DOC_ID, MODEL)
         │
         ↓
    合併所有結果
         │
         ↓
    輸出檔案
         ├─ CSV: data/extracts/{model_id}_extract_v1.0.csv
         └─ JSONL: data/outputs/{model_id}_extract_v1.0.jsonl
         │
         ↓
    前端顯示
         ├─ 進度條
         ├─ 成功/失敗數量
         └─ 結果預覽
```

### 流程 2：準確度分析 (Tab 2)

```
載入正確答案
    data/ground_truth/ground_truth.csv
         │
         ↓
    掃描萃取結果
         data/extracts/*_extract_*.csv
         │
         ↓
    用戶勾選檔案
         ☑ claude-sonnet-4-5@20250929_extract_v1.0.csv
         ☑ gemini-2.0-flash-001_extract_v1.0.csv
         │
         ↓
    點擊「開始分析」
         │
         ↓
    初始化 AccuracyAnalyzer
         │
         ↓
    載入並處理資料
         │
         ├─ 1. 讀取各模型 CSV
         ├─ 2. 正規化欄位格式
         ├─ 3. 統一日期/文字格式
         └─ 4. 以 CASE_LINK 為 key 合併
         │
         ↓
    計算準確度指標
         │
         ├─ NAME: Jaccard 相似度
         ├─ SEX: 精確匹配
         ├─ DATE_OF_BIRTH_YEAR: 年份匹配
         └─ PLACE_OF_BIRTH: Jaccard 相似度
         │
         ↓
    產生分析結果
         │
         ├─ 完全一致率
         ├─ 平均相似度
         └─ 完全不一致率
         │
         ↓
    視覺化
         │
         ├─ 圖表 1: 四欄位一致率比較 (長條圖)
         ├─ 圖表 2: 匹配情況統計 (堆疊圖 + 橫條圖)
         └─ 表格: Pivot Table (各模型各欄位準確度)
         │
         ↓
    提供下載
         ├─ accuracy_summary_{timestamp}.csv
         └─ detailed_data_{timestamp}.csv
```

## 📤 系統產出

### 1. 萃取結果檔案

**位置**: `data/extracts/`

**檔名格式**: `{model_id}_extract_v1.0.csv`

**內容範例**:
```csv
CASE_LINK,DOC_ID,MODEL,NAME,SEX,DATE_OF_BIRTH,PLACE_OF_BIRTH,...
local://file.pdf,file,claude-sonnet-4-5@20250929,Nguyễn Văn A,Nam,1990-01-01,Hà Nội,...
```

**欄位說明**:
- `CASE_LINK`: 案件連結或本地檔案路徑
- `DOC_ID`: 文件 ID
- `MODEL`: 使用的模型 ID
- `NAME`: 被告姓名
- `SEX`: 性別
- `DATE_OF_BIRTH`: 出生日期
- `DATE_OF_BIRTH_YEAR`: 出生年份
- `PLACE_OF_BIRTH`: 出生地
- 其他欄位...

### 2. JSONL 原始結果

**位置**: `data/outputs/`

**檔名格式**: `{model_id}_extract_v1.0.jsonl`

**內容**: 每行一個 JSON 物件，未經處理的原始萃取結果

### 3. 分析報告

**位置**: 使用者下載

**檔案**:
- `accuracy_summary_{timestamp}.csv` - 準確度摘要
- `detailed_data_{timestamp}.csv` - 完整比對資料

## 🔑 核心模組功能

### config.py - 配置管理

```python
職責:
├─ 定義路徑常數 (BASE_DIR, EXTRACT_DIR, etc.)
├─ 定義 ModelConfig 資料結構
├─ 提供參考模型清單 (REFERENCE_MODELS)
├─ 提供檔案命名函數 (get_extract_filename)
└─ 提供模型 ID 提取函數 (extract_model_id_from_filename)

關鍵函數:
├─ sanitize_model_id()          # 清理模型 ID 特殊字元
├─ get_extract_filename()       # 產生萃取結果檔名
├─ get_jsonl_filename()         # 產生 JSONL 檔名
└─ extract_model_id_from_filename()  # 從檔名提取模型 ID
```

### extractor.py - PDF 萃取器

```python
職責:
├─ 初始化 GCP 和 AI 客戶端
├─ 根據 provider 選擇 Gemini 或 Claude SDK
├─ 載入和解析 PDF 檔案
├─ 呼叫 LLM API 進行萃取
└─ 解析和驗證 JSON 回應

關鍵類別:
└─ PDFExtractor
    ├─ __init__()              # 初始化（接收 ModelConfig 和 gcp_project）
    ├─ call_llm()              # 路由到正確的 LLM
    ├─ _call_gemini()          # 呼叫 Gemini
    ├─ _call_claude()          # 呼叫 Claude
    ├─ extract_from_pdf()      # 萃取單一 PDF
    └─ batch_extract()         # 批次萃取

Provider 判斷邏輯:
├─ provider == "gemini" → 使用 genai.Client
└─ provider == "claude" → 使用 AnthropicVertex
```

### analyzer.py - 準確度分析器

```python
職責:
├─ 載入正確答案和模型結果
├─ 正規化和清理資料
├─ 計算各種相似度指標
└─ 產生分析報告

關鍵類別:
└─ AccuracyAnalyzer
    ├─ __init__()              # 初始化（接收 ground_truth_df）
    ├─ load_model_result()     # 載入模型結果
    ├─ merge_results()         # 合併多個模型結果
    ├─ calculate_accuracy()    # 計算準確度
    ├─ calculate_name_accuracy()  # 計算姓名準確度
    ├─ exact_match()           # 精確匹配
    ├─ year_match()            # 年份匹配
    └─ jaccard_similarity()    # Jaccard 相似度
```

## 💡 關鍵設計決策

### 1. 檔案命名 = 模型 ID
```
優勢: 追溯性強、不會混淆
範例: claude-sonnet-4-5@20250929_extract_v1.0.csv
```

### 2. Provider 自動判斷
```
根據模型名稱包含的關鍵字自動選擇 SDK
Gemini: gemini, palm, bison
Claude: claude, anthropic, sonnet, opus, haiku
```

### 3. 資料夾集中管理
```

├── extracts/       # 所有萃取結果
├── ground_truth/   # 正確答案

方便未來遷移到 GCS
```

### 4. 動態配置
```
前端輸入 → 即時配置 → 不依賴硬編碼
支援任意模型、專案、Location
```

## 🎯 總結

這是一個完整的 LLM 萃取準確度監控系統：

1. **萃取** - 使用 Gemini/Claude 從 PDF 萃取結構化資料
2. **儲存** - 以模型 ID 命名的 CSV 和 JSONL 檔案
3. **分析** - 與正確答案比對，計算多種準確度指標
4. **視覺化** - 完整的圖表和表格展示
5. **報告** - 可下載的 CSV 分析報告

系統特色：
- ✅ 完全動態化配置
- ✅ 支援多種 LLM 模型
- ✅ 智能 Provider 判斷
- ✅ 準備好遷移到 GCS
- ✅ 清晰的資料流向
- ✅ 易於擴展和維護
